Interactor

class Interactor<State, Event>(initialState: State, source: Flow<State>, coroutineScope: CoroutineScope, reducers: List<Reducer<State, Event>>, sideEffects: List<SideEffect<State, Event>>, stateResolver: (initial: State, source: State) -> State = { _, s -> s })

Coordinates an MVI loop by combining a source Flow of State with user-provided events.

Pipeline:

  1. A long-lived source flow provides upstream state (e.g., repository streams).

  2. Dispatched events are folded using reducers to produce the next State.

  3. After each reduction, sideEffects run (on Default dispatcher).

  4. The resulting stream is exposed as a hot state via stateIn bound to coroutineScope.

Parameters

initialState

The initial state value if the source has not emitted yet (and for restore).

source

The upstream State flow (e.g., from repositories). It can be a simple flow { emit(initialState) }.

coroutineScope

Scope used to host the resulting StateFlow; typically a lifecycle scope.

reducers

Ordered list of pure reducers applied to every event.

sideEffects

Ordered list of side-effects executed after state changes.

stateResolver

Given the previously saved initial state (e.g., from restore) and the latest emitted state from the source, return the merged state to start scanning from.
